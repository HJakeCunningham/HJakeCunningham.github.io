# HJakeCunningham.github.io

Personal academic website (static, hosted on GitHub Pages).

The page is **generated from data files** so you never have to touch HTML to add a
paper, job, or degree. You edit a small list in `data/`, run the build, and commit.

## How to update content

All content lives in `data/`:

| File                   | What it controls                                  |
| ---------------------- | ------------------------------------------------- |
| `data/site.js`         | Name, profile card, social links, intro bio       |
| `data/publications.js` | Publications list                                 |
| `data/experience.js`   | Work experience list                              |
| `data/education.js`    | Education list                                    |

Each list is just an array of entries shown top-to-bottom. To **add a paper**,
open `data/publications.js`, copy an existing block, and edit it:

```js
{
  title:   "My New Paper",
  authors: ["A Coauthor", "HJ Cunningham"], // your name is auto-underlined
  venue:   "Some Conference, 2026",
  image:   "./assets/images/my_thumbnail.png",
  links:   [{ label: "Paper", href: "https://arxiv.org/abs/..." }],
}
```

Adding experience or education works the same way (see the comments at the top of
each file). Put images in `assets/images/`, logos in `assets/logos/`, and PDFs in
`assets/papers/` or `assets/posters/`.

## Build & deploy

After editing data, regenerate the site:

```bash
npm install      # first time only
npm run build    # regenerates index.html AND rebuilds the CSS
git add -A && git commit -m "update content" && git push
```

GitHub Pages serves the committed `index.html` + `dist/output.css`.

### Useful scripts

- `npm run build` – regenerate `index.html` from data **and** build the CSS (run this before committing).
- `npm run build:html` – regenerate `index.html` only.
- `npm run dev` – regenerate `index.html` once, then watch and rebuild CSS on change.

> Note: `index.html` is generated by `build.js` — don't edit it by hand, your
> changes will be overwritten on the next build. Edit `data/` instead. The page
> layout/markup lives in `build.js` if you ever need to change the structure.
